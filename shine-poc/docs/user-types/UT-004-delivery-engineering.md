# Delivery & Engineering

## Overview

Delivery & Engineering users are technical practitioners who build solutions for clients. They frequently face problems that have been solved before somewhere in the organization. SHINE helps them discover existing code, patterns, and architectural decisions to avoid reinventing the wheel. They're sophisticated searchers who think in terms of technologies, frameworks, and design patterns.

## Key Workflows

1. **Pattern Discovery** - Search for existing implementations of common patterns (auth, data pipelines, APIs)
2. **Code Reuse** - Find, evaluate, and adapt existing code repositories
3. **Architecture Reference** - Review past architectural decisions and their outcomes
4. **Best Practice Lookup** - Find organizational standards and recommended approaches
5. **Contribution** - Submit improved patterns or new solutions back to the knowledge base

## Interaction Pattern

- **Entry Point:** Technical search (technology + pattern), or browsing by domain
- **Session Length:** 10-30 minutes (research phase)
- **Primary Actions:** Search, browse code, clone repos, bookmark, occasionally contribute
- **Preferred Output:** Code snippets, repo links, architecture diagrams, README files

## Success Metrics

| Metric | Target | Rationale |
|--------|--------|-----------|
| Code reuse instances | +25% YoY | Measure actual reuse |
| Time to find pattern | < 10 minutes | Efficient discovery |
| Contribution rate | > 10% of users | Healthy flywheel |
